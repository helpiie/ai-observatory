# StratoAtlas Pressure Signal Reference

## Runtime Authorization History Does Not Prove Runtime Control

This repository is connected to a published named contribution to StratoAtlas:

- Runtime Authorization History Does Not Prove Runtime Control  
  https://stratoatlas.com/pressure/runtime-authorization-history-does-not-prove-control

The contribution focuses on runtime authorization, authority-state continuity, and evidence of control in high-consequence AI and operational systems.

## Core distinction

Authorization history can show that participation, approval, or delegation occurred.

It does not necessarily prove that control remained valid at the moment of execution.

The central distinction is:

> Approval is an event. Control is a runtime relationship.

A system may preserve a record of authorization while failing to prove that the valid authority relationship remained connected to the executed action.

## Four-layer distinction

The contribution distinguishes between:

1. Declarative control  
   The stated authority relationship.

2. Authority object  
   The scope, mandate, or envelope that was authorized.

3. Runtime-validity object  
   The active evaluation structure comparing the current execution state against the conditions under which the original authorization remains executable.

4. Evidence of control  
   Observable proof that the valid authority relationship remained causally connected to the executed action.

## Compact diagnostic

> If the system cannot compare the current execution state with the validity conditions of the approved envelope, it can show authorization history, but not runtime control.

## Related second-order pressure

A related second-order StratoAtlas record develops the pressure further:

- A Runtime-Validity Object Does Not by Itself Establish Runtime Control  
  https://stratoatlas.com/pressure/runtime-validity-object-does-not-establish-control

That second-order pressure emphasizes that a runtime-validity object only supports runtime control if it can operate against maintained validity conditions, represent current state, handle uncertainty, causally constrain execution, avoid self-attestation, and preserve meaningful human intervention capacity.
